Reinier Ladan de Front-end Developer

Naast interactie-ontwerper ben ik ook front-end developer. Dit is handig in scrumprojecten omdat ik kan meedraaien met ontwerp én techniek. Daarnaast ben ik ook in te zetten als dedicated front-end developer zonder dat ik ook maar één pixel hoef te ontwerpen. Die dingen kan ik prima gescheiden houden.

Check mijn Github profiel voor voorbeelden van mijn front-end werk:

Github Reinier Ladan

{% markdown %} ----------------------- Een paar richtlijnen qua front-end werkzaamheden: - Ik werk het liefst met plain HTML5, maar iets als [HAML](http://haml.info) is ook geen probleem (al zorgt [Emmet](http://docs.emmet.io) ervoor dat ik 'normale' HTML ook vrij snel kan typen). - Ik werk het liefst met [SASS](http://sass-lang.com), maar [LESS](http://lesscss.org) of plain-old CSS is ook geen probleem. - Ik werk het liefst met [jQuery](http://jquery.com) of [Underscore.js](http://underscorejs.org) als het om JavaScript gaat, maar plain JavaScript behoort ook tot de mogelijkheden. - Ik werk het liefst op mijn eigen apparatuur ([MacBook Pro](http://www.apple.com/macbook-pro/)) en met [Sublime Text 3](http://www.sublimetext.com/3) en command line tools zoals Git en Grunt/Gulp maar bij hoge uitzondering kan ik ook op door de klant verzorgde apparatuur werken (waarschijnlijk ivm een speciaal systeem dat voor het werk nodig is). Graag van tevoren overleg indien dit nodig is. - Ik zorg ervoor dat wat ik maak werkt op de laatste twee versies van de grote browsers (Internet Explorer, Chrome, Safari en Firefox). Over ondersteuning voor andere browsers en versies kan uiteraard gepraat worden. - Ik zorg ervoor dat *prototypes* werken in Chrome en/of Safari. Meer browserondersteuning voor een prototype haalt het nut van een prototype onderuit. De ondersteuning voor Chrome/Safari zorgt ervoor dat op de meeste platformen de mogelijkheid bestaat het prototype te bekijken. - Ik heb ervaring met de volgende backend frameworks en CMS'en: Laravel, Wordpress, Movable Type, Ruby on Rails, Silex, CodeIgniter, CakePHP en Symphony — mijn ervaring leert dat ik het verwerken van front-end code snel oppak in een willekeurig framework en verschillende template engines. ----------------------- Mocht ik nou helemaal vol zitten met werk, kan gebeuren, dan kun je ook terecht bij: - [Peter van Grieken](http://petervangrieken.nl) - [Wietse Hage](http://wietsehage.nl) - [Charis Rooda](http://www.imakewebsites.nl) - [Hidde de Vries](http://hiddedevries.nl) - [Jeroen Hulscher / eBrella](http://www.ebrella.com) {% endmarkdown %}
